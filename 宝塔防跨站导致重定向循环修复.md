# 後台重定向循環 / ERR_TOO_MANY_REDIRECTS 修復

## 原因

**防跨站攻擊 (open_basedir)** 開啟時，PHP 僅能訪問網站目錄（如 `public/`）。  
ThinkPHP 需要讀取 `application/` 下的 `install.lock`，該目錄在 `public/` 外，被 open_basedir 限制，導致 `is_file()` 失敗，不斷重定向到 `install.php`。

## 修復方式

### 方式一：關閉防跨站（建議）

1. 宝塔 → 網站 → MXTRX.TOP → 設置
2. 網站目錄
3. 將 **防跨站攻擊** 關閉（開關切為關閉）
4. 保存

### 方式二：修改 PHP 配置

若必須保持防跨站，需將上級目錄納入允許範圍：

1. 网站設置 → PHP → 禁用函数（或 PHP 配置文件）
2. 找到 `open_basedir`，改為：
   ```
   /www/wwwroot/old/backend-php/:/tmp/
   ```
3. 重載 PHP
