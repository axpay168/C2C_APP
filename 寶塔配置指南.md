# 寶塔面板配置指南

## 一、創建網站

1. 登入寶塔面板
2. 點擊「網站」→「添加站點」
3. 填寫以下信息：
   - **域名**: `mxtrx.top` 和 `www.mxtrx.top`（根據實際域名修改）
   - **根目錄**: `/www/wwwroot/old/backend-php/public`
   - **PHP 版本**: 選擇 PHP 8.2
   - **網站分類**: 可選

## 二、配置 Nginx

1. 在網站列表中，找到剛創建的網站，點擊「設置」
2. 點擊「配置文件」標籤
3. 將以下配置內容替換到配置文件中：

```nginx
server {
    listen 80;
    server_name mxtrx.top www.mxtrx.top;
    
    # 網站根目錄
    root /www/wwwroot/old/backend-php/public;
    index index.php index.html index.htm;
    
    # 前端 H5 - 靜態文件
    location /h5/ {
        try_files $uri $uri/ /h5/index.html;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }
    
    # 後端 API 與 PHP
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    
    location ~ \.php$ {
        fastcgi_pass unix:/tmp/php-cgi-82.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_param PATH_INFO $fastcgi_path_info;
    }
    
    # 靜態資源緩存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff|woff2|ttf|svg)$ {
        expires 30d;
        access_log off;
    }
    
    location ~ /\. {
        deny all;
    }
    
    access_log /www/wwwlogs/mxtrx.top.log;
    error_log /www/wwwlogs/mxtrx.top.error.log;
}
```

4. 點擊「保存」

## 三、設置偽靜態（可選）

如果需要更好的 URL 路由，可以在「偽靜態」標籤中添加：

```nginx
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php?s=$1 last;
        break;
    }
}
```

## 四、PHP 設置

1. 點擊「PHP 版本」標籤
2. 確保選擇 PHP 8.2
3. 點擊「設置」→「擴展」→ 確保以下擴展已啟用：
   - pdo_mysql
   - curl
   - json
   - mbstring
   - fileinfo（如果可用）

## 五、測試訪問

配置完成後，訪問以下地址進行測試：

1. **測試腳本**: `http://您的域名/test.php`
2. **後台入口**: `http://您的域名/buSRxMqJOo.php` 或 `http://您的域名/index.php/admin/index/login`
3. **H5 前端**: `http://您的域名/h5/`
4. **API 測試**: `http://您的域名/index.php/api/index/index`

## 六、SSL 證書（可選）

如果需要 HTTPS：
1. 在網站設置中點擊「SSL」
2. 選擇「Let's Encrypt」免費證書
3. 點擊「申請」
4. 申請成功後，將 HTTP 重定向到 HTTPS

## 七、常見問題

### 1. 404 錯誤
- 檢查根目錄是否正確指向 `/www/wwwroot/old/backend-php/public`
- 檢查 Nginx 配置是否正確保存並重載

### 2. 500 錯誤
- 檢查 PHP 錯誤日誌：`/www/wwwlogs/mxtrx.top.error.log`
- 檢查文件權限是否正確

### 3. 數據庫連接失敗
- 檢查數據庫用戶名和密碼是否正確
- 檢查數據庫是否已導入
