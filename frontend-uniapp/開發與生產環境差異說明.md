# 開發與生產環境說明

## 使用當前生產環境頁面進行開發（已同步）

- 開發端靜態包（`static-h5/`、`preview-prod/h5/`、`static-h5-serve/h5/`）已改為**從生產環境複製**的內容，與 **https://mxtrx.top/h5/** 一致。
- **生產環境未被修改**：僅讀取 `backend-php/public/h5/` 並覆蓋到上述開發目錄，未對生產代碼或運行做任何改動。
- **編譯產物說明**：生產環境為已編譯的靜態檔（minified JS/CSS），且**無 source map**，無法還原成 `.vue` 源碼；開發時是以「與生產相同的靜態包」做預覽與除錯，若要改頁面邏輯需在專案內 `pages/`、`src/` 等源碼處修改後再 `npm run build:h5` 部署。
- **語言顯示與生產不一致**：頁面語言由 **localStorage 的 `lang`** 決定（見 `src/common/locales/config.js`），與靜態包是否覆蓋無關。localhost 與 mxtrx.top 是不同網域，localStorage 互不共用，所以開發端可能顯示韓文/中文、生產端顯示英文。若要與生產畫面一致，請在本地清除該站點儲存（或刪除 `lang`）後重新整理，或於頁面內切換語言為英文。

## 讓開發環境與生產環境顯示一致（且不影響生產）

- **開發環境**：執行 `npm run dev`，會先將目前生產用的靜態包（static-h5）拷貝到 preview-prod/h5，並在 **8083** 端口以路徑 **/h5/** 提供服務。
- **訪問方式**：瀏覽器打開 **http://你的機器IP:8083/h5/**（本機瀏覽器請用伺服器 IP，勿用 localhost）。
- **同步**：若生產有更新，可執行 `npm run sync:from-prod` 覆蓋 `static-h5/`，再執行 `npm run dev` 即可看到與生產相同內容。
- **不影響生產**：以上僅使用 static-h5 / preview-prod，不會對 mxtrx.top 上的代碼或靜態檔做任何改動。

| 指令 | 說明 |
|------|------|
| `npm run dev` | 與生產一致的靜態預覽（埠 **8083**，路徑 /h5/） |
| `npm run preview:prod:8083` | 同上 |
| `npm run sync:from-prod` | 從生產拉取 h5 到 static-h5 |

### 與生產「完全一致」的預覽方式

1. 執行 **`npm run dev`** 或 **`npm run preview:prod:8083`**。
2. 瀏覽 **http://你的伺服器IP:8083/h5/**（防火牆已開放 8083）。
3. 若要語言與生產一致：F12 → Application → Local Storage，清除該站點的 `lang` 後重新整理。

### 覆蓋生產包後沒有變化？排查步驟

1. **看腳本檔名**：DevTools → Elements，生產靜態包會載入 `chunk-vendors.09bcdb88.js`、`index.5de6bcdd.js`（帶 hash）。
2. **強制重新載入**：Ctrl+Shift+R 或清除該站點快取。
3. **語言仍不同**：清除 localStorage 的 `lang` 或於頁面內切換為英文。

# 為什麼開發與生產展示不一樣？

## 原因說明

### 1. 兩邊的「內容來源」不同

| 環境 | 內容來源 | 何時更新 |
|------|-----------|----------|
| **生產** `https://mxtrx.top/h5/` | `backend-php/public/h5/` 裡的**靜態檔** | 執行 `npm run deploy:h5` 時才會變 |
| **開發** `http://伺服器IP:8083/h5/` | `preview-prod/h5/`（由 static-h5 同步，與生產同包） | 執行 `npm run sync:from-prod` + `npm run dev` 時更新 |

### 2. 生產環境實際用的是哪一包？

- 載入的是 `index.5de6bcdd.js`、`chunk-vendors.09bcdb88.js`。
- 開發 8083 預覽與生產使用同一份 static-h5 打包。

---

## 要怎麼讓兩邊一樣？

### 做法一：讓「生產 = 現在開發的程式碼」（建議）

1. 在專案根目錄執行：
   ```bash
   npm run build:h5
   npm run deploy:h5
   ```
2. 這樣會用**目前的** `pages/common/login.vue` 等源碼打包，並覆蓋 `backend-php/public/h5/`。
3. 之後 **mxtrx.top/h5/** 就會和你在 **localhost 上跑的 dev 畫面一致**（同一份源碼、同一份打包）。

### 做法二：開發時想看和生產「完全同一包」畫面

- 執行 `npm run dev` 或 `npm run start`，在埠 8083 或 8082 預覽 static-h5，與生產畫面一致。

---

## 總結

- 開發預覽用 **8083**（`npm run dev`），與生產使用同一份 static-h5 打包。
- 若要部署新源碼到生產：`npm run build:h5` + `npm run deploy:h5`。
