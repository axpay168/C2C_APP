# HBuilderX H5 預覽登入頁面問題解決方案

## 問題診斷

### 問題描述
HBuilderX 編譯成功，但預覽時沒有顯示登入頁面。

### 根本原因分析

1. **入口頁面配置問題**
   - `pages.json` 中第一個頁面是 `pages/index/index`（首頁），而不是登入頁面
   - H5 預覽會直接進入首頁，而不是登入頁面

2. **首頁載入時出錯**
   - 在 `onLoad()` 中調用了 `getNocar()`，訪問 `res.data[0]` 沒有檢查
   - 如果 API 返回數據不完整，會導致運行時錯誤
   - 錯誤可能阻止頁面正常載入，無法執行到 `checkIsLogin()`

3. **登入檢查時機問題**
   - `checkIsLogin()` 在 `onShow()` 中執行
   - 如果首頁在 `onLoad()` 時就出錯，可能無法執行到 `onShow()`

4. **H5 路由配置**
   - `manifest.json` 中配置了 `"base": "h5/"`
   - 這可能導致路由路徑問題

---

## 解決方案

### 方案一：將登入頁面設為入口頁面（推薦）

修改 `pages.json`，將登入頁面移到第一位：

```json
{
  "pages": [{
    "path": "pages/common/login",  // 移到第一位
    "style": {
      "navigationBarTitleText": "",
      "enablePullDownRefresh": false
    }
  }, {
    "path": "pages/index/index",  // 原首頁
    "style": {
      "navigationBarTitleText": "",
      "enablePullDownRefresh": false
    }
  },
  // ... 其他頁面
  ]
}
```

**優點**：
- 直接顯示登入頁面
- 符合用戶預期
- 簡單直接

**缺點**：
- 需要修改頁面順序

---

### 方案二：在首頁 onLoad 中檢查登入（已實施）

在首頁 `onLoad()` 方法開始時就檢查登入狀態：

```javascript
onLoad(options) {
  // 先檢查登入狀態
  const token = uni.getStorageSync('token') || ""
  if (!token) {
    uni.redirectTo({
      url: "/pages/common/login"
    })
    return  // 立即返回，不執行後續代碼
  }
  // ... 其他初始化代碼
}
```

**優點**：
- 不需要修改 pages.json
- 保持原有頁面結構
- 確保未登入用戶立即跳轉

**缺點**：
- 會先載入首頁，然後跳轉（可能有短暫閃爍）

---

### 方案三：修復首頁載入錯誤（已實施）

修復 `getNocar()` 方法中的數組訪問問題：

```javascript
getNocar() {
  const token = uni.getStorageSync('token')
  this.$u.api.index.get_placard(token).then(res => {
    // 安全訪問數組元素
    if (res && res.data && Array.isArray(res.data) && res.data.length > 0 && res.data[0] && res.data[0].titles) {
      this.news = ["" + res.data[0].titles + ""]
    }
  }).catch(err => {
    console.warn('getNocar failed:', err)
  })
}
```

**優點**：
- 防止運行時錯誤
- 確保頁面能正常載入
- 即使 API 失敗也不會崩潰

## 測試步驟

1. **清除瀏覽器緩存**
   - 按 `Ctrl + Shift + Delete` 清除緩存
   - 或使用無痕模式測試

2. **在 HBuilderX 中重新運行**
   - 停止當前的運行
   - 點擊「運行 → 運行到瀏覽器 → Chrome」
   - 等待編譯完成

3. **檢查預覽結果**
   - 應該直接顯示登入頁面（如果未登入）
   - 或顯示首頁（如果已登入）

4. **檢查控制台**
   - 按 `F12` 打開開發者工具
   - 查看 Console 標籤
   - 確認沒有錯誤訊息

---

## 如果問題仍然存在

### 檢查項目

1. **檢查 HBuilderX 運行配置**
   - 工具 → 設置 → 運行配置
   - 確認 Node.js 和 npm 路徑正確

2. **檢查編譯輸出**
   - 查看 `unpackage/dist/build/web/index.html`
   - 確認文件存在且完整

3. **檢查瀏覽器控制台**
   - 查看具體錯誤訊息
   - 檢查 Network 標籤，確認資源載入正常

4. **嘗試方案一**
   - 將登入頁面設為入口頁面
   - 這是最直接的解決方案

## 最佳實踐建議

1. **入口頁面選擇**
   - 對於需要登入的應用，建議將登入頁面設為入口頁面
   - 或使用方案二，在首頁立即檢查登入狀態

2. **錯誤處理**
   - 所有 API 調用都應該有錯誤處理
   - 訪問數組或對象屬性前，先檢查是否存在

3. **登入檢查時機**
   - 在 `onLoad()` 中檢查，而不是 `onShow()`
   - 確保在頁面載入前就進行檢查
