# H5 編譯錯誤修復說明

## 已修復：UNI_CLI_CONTEXT 錯誤

**錯誤**：`TypeError [ERR_INVALID_ARG_TYPE]: The "paths[0]" argument must be of type string. Received undefined`

**原因**：`process.env.UNI_CLI_CONTEXT` 在 plugin 初始化時未設定。

**修復**：在執行 build 前設定環境變數，`package.json` 已更新為：

```json
"build:h5": "cross-env NODE_ENV=production UNI_PLATFORM=h5 UNI_CLI_CONTEXT=$(pwd) UNI_INPUT_DIR=$(pwd) vue-cli-service build"
```

**注意**：`$(pwd)` 在 Windows 可能無效，Linux/Mac 可用。若在 Windows，可改為：

```bash
set UNI_CLI_CONTEXT=%cd% && set UNI_INPUT_DIR=%cd% && npm run build:h5
```

或直接執行：

```bash
cd /www/wwwroot/old/frontend-uniapp
UNI_CLI_CONTEXT=$(pwd) UNI_INPUT_DIR=$(pwd) npm run build:h5
```

---

## 已修復：Webpack 版本衝突

**錯誤**：`configuration.module.rules[5] has an unknown property 'generator'`、`moduleIds` 無效

**原因**：`@vue/cli-plugin-babel@5` 與 `@vue/cli-service@4` 不兼容，導致 Webpack 5 與 4 衝突。

**修復**：已將 `@vue/cli-plugin-babel` 降級為 `^4.5.19`。

## 尚待處理：recyclableRender 錯誤

**錯誤**：`Export 'recyclableRender' is not defined`（出現在 App.vue、fundRecord.vue）

**原因**：uni-app 的 vue-loader 與 vue-template-compiler 版本不完全兼容，導致 template 編譯產物不一致。

**建議**：優先使用 **HBuilderX** 編譯（見下方）

---

## 備選：使用 HBuilderX 編譯

若 npm build 仍有問題，建議使用 **HBuilderX**：

1. 用 HBuilderX 開啟 `/www/wwwroot/old/frontend-uniapp`
2. 選單「發行」→「網站-H5」
3. 輸出到 `unpackage/dist/build/h5/`
4. 再複製到 `/www/wwwroot/old/backend-php/public/h5/`
