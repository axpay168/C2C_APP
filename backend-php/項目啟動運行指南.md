## 一、環境要求

- ✅ PHP 8.2+（已確認安裝）
- ✅ MySQL 5.7+
- ✅ Nginx 或 Apache
- ✅ PHP 擴展：pdo_mysql、curl、json、mbstring、openssl

---

## 二、啟動前檢查

### 1. 執行啟動檢查腳本

```bash
cd /www/wwwroot/hkex-sm.com
chmod +x 啟動项目.sh
./启动项目.sh
```

### 2. 確認服務運行

在寶塔面板或命令行確認：

| 服務 | 檢查命令 | 說明 |
|------|----------|------|
| MySQL | `systemctl status mysql` | 需為運行中 |
| PHP-FPM | `systemctl status php-fpm-82` | 或對應版本 |
| Nginx | `systemctl status nginx` | 或 Apache |

### 3. 目錄權限

確保以下目錄可寫（PHP-FPM 用戶為 www）：

```bash
chmod -R 777 /www/wwwroot/hkex-sm.com/runtime
chmod -R 777 /www/wwwroot/hkex-sm.com/public/uploads
```

---

## 三、Web 服務器配置（重要）

### Nginx 配置要點

網站根目錄 **必須** 指向 `public` 目錄：

```nginx
root /www/wwwroot/hkex-sm.com/public;
index index.php index.html index.htm;

location / {
    try_files $uri $uri/ /index.php?$query_string;
}

location ~ \.php$ {
    fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 按實際 PHP 版本調整
    fastcgi_index index.php;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    include fastcgi_params;
}
```

### 寶塔面板操作

1. 網站 → 選擇對應網站 → 設置
2. 網站目錄 → 運行目錄選擇 `/public`
3. 偽靜態：選擇 `thinkphp` 或自定義

## 六、常見問題

### 502 Bad Gateway

- 檢查 PHP-FPM 是否運行
- 檢查 Nginx 中 `fastcgi_pass` 的 sock 路徑是否正確

### 404 Not Found

- 確認網站根目錄為 `/www/wwwroot/hkex-sm.com/public`
- 確認偽靜態規則已啟用

### 數據庫連接失敗

- 確認 MySQL 已啟動
- 確認數據庫 hkex02 已創建並導入
- 檢查 `application/database.php` 中的賬號密碼

### 客戶端 API 調用失敗

- 客戶端 API 地址在 `數據庫前端/ouyuan 2/common/http.interceptor.js`
- 確保 `baseUrl` 為您的實際域名，例如：`http://您的域名/api`
