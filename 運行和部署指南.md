# old 專案 - 運行和部署指南

## 前端：靜態獨立運行（不依賴 HBuilderX）

前端**優先以靜態方式**開啟，方便本地測試與完善，無需 HBuilderX。

### 方式一：直接啟動靜態 H5（推薦，立即可用）

已內建一份編譯好的靜態 H5（來自 hkex，在 `static-h5/`），可直接用 npm 啟動：

```bash
cd /www/wwwroot/old/frontend-uniapp

# 若尚未安裝依賴，先執行（僅需一次）
npm install --legacy-peer-deps

# 啟動靜態服務，預設埠 8082
npm run start
# 或
npm run start:static
# 或
npm run static
```

瀏覽器訪問：**http://localhost:8082**

- 無需 HBuilderX
- 無需先編譯，適合先做頁面與流程測試
- 修改後端 API 或介面邏輯時，可之後再用「方式二」編譯

### 方式二：從源碼編譯後再以靜態開啟

需要改動 Vue 源碼並看效果時：

```bash
cd /www/wwwroot/old/frontend-uniapp
npm install --legacy-peer-deps
npm run dev:h5
# 開發服務預設在 http://localhost:8080（見 vue.config.js）
```

或編譯生產版再靜態開啟：

```bash
npm run build:h5
npm run serve:dist
# 會對 unpackage/dist/build/h5 啟動靜態服務，埠 8082
```

### 方式三：HBuilderX（可選）

若仍想用 HBuilderX：打開 `old/frontend-uniapp` → 運行到瀏覽器 / 發行 H5。  
發行完成後可執行 `npm run deploy:h5` 將輸出複製到後端 `public/h5/`。

### 腳本一覽

| 腳本 | 說明 |
|------|------|
| `npm run start` / `start:static` / `static` | 用 `static-h5/` 啟動靜態服務（埠 8082） |
| `npm run dev:h5` | 開發模式（需已安裝依賴） |
| `npm run build:h5` | 編譯 H5 到 `unpackage/dist/build/h5` |
| `npm run serve:dist` | 對編譯結果啟動靜態服務（埠 8082） |
| `npm run deploy:h5` | 將 H5 複製到 `../backend-php/public/h5/` |

## 後端 (PHP)

```bash
cd /www/wwwroot/old/backend-php

# 安裝依賴（需 PHP 7.2+，且啟用 ext-fileinfo、ext-curl、ext-pdo、ext-bcmath）
composer install
# 若環境無 composer 或擴展不全，可在本機執行後再上傳 vendor
```

配置數據庫：編輯 `application/database.php`（庫名、用戶名、密碼、前綴 `fa_`）。

導入數據庫：

```bash
mysql -u root -p 數據庫名 < /www/wwwroot/old/db/schema_full.sql
```

Nginx 站點根目錄指向 `old/backend-php/public`。
