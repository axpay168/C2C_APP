# H5 前端功能測試指南

## 📋 測試概述

本次測試主要驗證 H5 前端功能是否正常，特別是登入和註冊按鈕的修復是否生效。

## ✅ 已完成的修復

1. **按鈕修復**：
   - 將登入和註冊按鈕從 `<p>` 標籤改為 `<view>` 標籤
   - 將 `@click` 事件改為 `@tap` 事件（H5 環境兼容）
   - 添加 `cursor: pointer` 樣式

2. **編譯狀態**：
   - ✅ H5 編譯成功
   - ✅ 編譯產物已複製到 `/www/wwwroot/old/backend-php/public/h5/`
   - ✅ 發現登入相關 JS 文件：
     - `static/js/pages-common-login.c45bfbdd.js`
     - `static/js/pages-common-login.5b977708.js`
     - `static/js/pages-common-register.ce467a2e.js`

2. **Network 標籤**：

## 🐛 常見問題排查

### 問題 1：按鈕仍然無反應

**可能原因**：
- 瀏覽器緩存未清除
- 編譯產物未正確複製
- JavaScript 文件未正確加載

**解決方法**：
1. 清除瀏覽器緩存（Ctrl+Shift+Delete）
2. 硬刷新頁面（Ctrl+F5）
3. 檢查瀏覽器控制台是否有錯誤
4. 確認 JS 文件是否正常加載

### 問題 2：API 請求失敗

**可能原因**：
- Nginx 配置未正確應用
- API 路徑錯誤
- CORS 問題

**解決方法**：
1. 檢查 Nginx 配置是否正確
2. 檢查 API 基礎路徑是否正確
3. 檢查後端服務是否正常運行

### 問題 3：樣式顯示異常

**可能原因**：
- CSS 文件未正確加載
- 路徑配置錯誤

**解決方法**：
1. 檢查 Network 標籤中的 CSS 文件加載狀態
2. 檢查文件路徑是否正確
