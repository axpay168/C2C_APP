# 前端登入測試說明

## 問題與修復

### 1. 登入頁 setDefaultLang 錯誤
- **原因**：Store 預設 `lang` 為 `en`，但語言列表只有 `eng`，`findIndex` 回傳 -1 導致崩潰
- **修復**：已在 `pages/common/login.vue` 與建置檔 `pages-common-login.c45bfbdd.js` 加入 `n<0` 檢查
- **源碼**：已將 store 預設改為 `eng`

### 2. API 請求仍指向 hkex-sm.com
- **現象**：部分請求送往 `https://hkex-sm.com`，無法解析
- **解法**：在本機加入 hosts，讓瀏覽器解析 hkex-sm.com 到測試伺服器

## 測試步驟

### 方法一：使用 hosts（建議）

1. **在本機編輯 hosts**：
   ```bash
   # Linux/Mac: /etc/hosts
   # Windows: C:\Windows\System32\drivers\etc\hosts
   
   149.104.29.96 hkex-sm.com www.hkex-sm.com
   ```

2. **用網域開啟頁面**：
   ```
   http://hkex-sm.com/h5/
   ```

3. 登入資訊：
   - 帳號：`testuser@hkex-test.com`
   - 密碼：`Test123456`

### 方法二：直接用 IP 訪問

1. 進入登入頁：`http://149.104.29.96/h5/#/pages/common/login`
2. 強制重新整理（Ctrl+Shift+R）清除快取
3. 使用上述帳密登入

### 方法三：透過 API 取得 Token 後直接進入首頁

```bash
# 1. 取得 token
TOKEN=$(curl -s -X POST "http://149.104.29.96/index.php/api/user/login" \
  -d "account=testuser@hkex-test.com&password=Test123456&lang=eng" \
  -H "Content-Type: application/x-www-form-urlencoded" | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 2. 開啟瀏覽器 Console，執行：
# uni.setStorageSync('token', '你的TOKEN');
# 然後導航到 http://149.104.29.96/h5/#/pages/index/index
```

## Nginx 設定

若尚未更新，請將 `nginx-hkex-ip.conf` 複製到：
```bash
cp /www/wwwroot/hkex-sm.com/nginx-hkex-ip.conf /www/server/panel/vhost/nginx/
nginx -t && nginx -s reload
```
