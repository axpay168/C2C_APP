# 启动MySQL服务指南

## 🔍 问题诊断

测试结果显示：**数据库连接失败**

错误信息：`SQLSTATE[HY000] [2002] Unknown error while connecting`

这通常表示：**MySQL服务未运行**

## 🚀 解决方案：在宝塔面板中启动MySQL服务

### 第一步：检查MySQL服务状态

1. **登录宝塔面板**
2. **点击左侧菜单"软件商店"**
3. **找到"MySQL"**（可能是 MySQL 5.7、MySQL 8.0 或其他版本）
4. **点击右侧的"设置"按钮**
5. **进入"服务"标签**
6. **查看服务状态**：
   - ✅ **运行中** - MySQL服务正常
   - ❌ **已停止** - 需要启动MySQL服务

### 第二步：启动MySQL服务

如果MySQL服务状态为"已停止"：

1. **点击"启动"按钮**
2. **等待启动完成**（通常需要几秒钟）
3. **确认状态变为"运行中"**

### 第三步：验证MySQL服务

启动后，可以通过以下方式验证：

#### 方法1：在宝塔面板中查看

- MySQL服务状态显示为"运行中"
- 可以正常访问phpMyAdmin

#### 方法2：访问测试文件

在浏览器中访问：
```
http://MXTRX.TOP/test_db.php
```

如果MySQL服务已启动，应该能看到：
- ✅ 数据库连接成功
- ✅ FastAdmin表数量
- ✅ 管理员账号信息

#### 方法3：在phpMyAdmin中测试

1. **进入"数据库"管理**
2. **找到数据库 `hkex02`**
3. **点击"管理"或"phpMyAdmin"**
4. **如果能正常打开，说明MySQL服务正常**

## 🔧 如果MySQL服务无法启动

### 问题1：启动失败

**可能原因**：
- MySQL配置文件错误
- 端口被占用
- 数据文件损坏

**解决方法**：
1. 查看MySQL错误日志
2. 检查MySQL配置文件
3. 重启服务器（最后手段）

### 问题2：启动后立即停止

**可能原因**：
- MySQL配置文件有错误
- 数据目录权限问题
- 磁盘空间不足

**解决方法**：
1. 查看MySQL错误日志
2. 检查磁盘空间：`df -h`
3. 检查MySQL数据目录权限

### 问题3：无法连接MySQL

**可能原因**：
- MySQL服务运行但端口未开放
- 防火墙阻止连接
- MySQL用户权限问题

**解决方法**：
1. 检查MySQL端口（默认3306）是否开放
2. 检查防火墙设置
3. 检查MySQL用户权限
