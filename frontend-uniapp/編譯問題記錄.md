# old 專案編譯問題記錄

> 避免重複犯錯，遇問題先記錄再解決

## 已記錄問題

### 問題 1：npm / Node 未安裝或不在 PATH
- **現象**：執行 `npm install` 或 `npm run build:h5` 時報錯「無法辨識 npm」
- **原因**：系統未安裝 Node.js，或未加入環境變數 PATH
- **避免**：不依賴 npm 時，改用 HBuilderX 或 static-h5

### 問題 2：HBuilderX 創建的專案用 npm 編譯有 recyclableRender 錯誤
- **現象**：`Export 'recyclableRender' is not defined`
- **原因**：uni-app vue-loader 與 vue-template-compiler 版本不完全兼容
- **解決**：優先使用 HBuilderX 編譯，或使用已編譯的 static-h5

### 問題 3：UNI_CLI_CONTEXT 在 Windows 無效
- **現象**：`$(pwd)` 在 Windows CMD/PowerShell 無法正確解析
- **解決**：用 `%cd%` (CMD) 或 `$PWD` (PowerShell) 替代

### 問題 4：HBuilderX 安裝路徑不固定
- **現象**：cli.exe 位置需手動指定
- **常見路徑**：`C:\Program Files\HBuilderX`、`D:\HBuilderX`、解壓即用的目錄

### 問題 5：HBuilderX 編譯時找不到 npm 依賴（echarts、ethers、tronweb）✅ 已解決
- **現象**：`文件查找失敗：'echarts' at main.js:20`、`文件查找失敗：'ethers' at pages\index\index.vue:25`、`文件查找失敗：'tronweb' at pages\index\index.vue:524`
- **原因**：`node_modules` 目錄不存在，npm 依賴未安裝
- **解決方案**：
  1. **方案 A（推薦）**：在 HBuilderX 中安裝依賴
     - 開啟 HBuilderX
     - 選單：工具 → 插件安裝 → npm 安裝（或直接在專案根目錄執行 npm install）
     - 或在 HBuilderX 終端中執行：`npm install`
  2. **方案 B**：手動安裝依賴
     - 找到 Node.js 安裝路徑（通常在 `C:\Program Files\nodejs\` 或 `C:\Users\用戶名\AppData\Roaming\npm`）
     - 在專案目錄執行：`完整路徑\npm.cmd install`
     - 或將 Node.js 加入系統 PATH 環境變數
  3. **方案 C**：使用 HBuilderX 內建 Node.js
     - HBuilderX 通常自帶 Node.js，檢查：`HBuilderX安裝目錄\plugins\node\node.exe`
     - 使用該 Node.js 執行 npm install
- **解決狀態**：✅ 已通過方法二（HBuilderX 終端執行 npm install）成功解決
- **避免**：編譯前先確認 `node_modules` 目錄存在，或使用已編譯的 `static-h5`

### 問題 6：編譯時的 CSS 警告（非致命）
- **現象**：編譯時出現 postcss-loader 警告：
  - `end value has mixed support, consider using flex-end instead`
  - `You should write display: flex by final spec instead of display: box`
- **原因**：CSS 使用了舊的語法或兼容性寫法
- **影響**：⚠️ 僅為警告，不影響編譯和功能
- **解決**：可選修復，將舊 CSS 語法更新為現代標準（非必須）
- **避免**：這些警告可以忽略，不影響專案運行

### 問題 7：Vue 模板錯誤 - :value 與 v-model 衝突 ✅ 已解決
- **現象**：編譯時出現錯誤：
  - `:value="xxx" conflicts with v-model on the same element because the latter already expands to a value binding internally`
- **原因**：在同一個 input 元素上同時使用了 `:value` 和 `v-model`
  - `v-model` 實際上就是 `:value` 和 `@input` 的語法糖，不能同時使用
- **影響**：❌ 編譯失敗，無法運行
- **解決方案**：
  1. **對於可編輯的 input**：只使用 `v-model`，移除 `:value`
     ```vue
     <!-- 錯誤 -->
     <input :value="nickname" v-model="nickname">
     
     <!-- 正確 -->
     <input v-model="nickname">
     ```
  2. **對於只讀的 input**：只使用 `:value`，移除 `v-model`
     ```vue
     <!-- 錯誤 -->
     <input readonly :value="shijia" v-model="shijia">
     
     <!-- 正確 -->
     <input readonly :value="shijia">
     ```
- **已修復的文件**：
  - `pages/setting/info.vue` - nickname, password
  - `pages/index/financial.vue` - password, nickname, hhunye
  - `pages/index/sell.vue` - shijia, youjia, maijia
  - `pages/index/buy.vue` - shijia, youjia, maijia
- **避免**：不要在同一个元素上同时使用 `:value` 和 `v-model`

### 問題 8：運行時錯誤 - Cannot read properties of undefined (reading '0') ✅ 已解決
- **現象**：控制台每秒出現一次 TypeError：
  - `Cannot read properties of undefined (reading '0')`
- **原因**：在 `pages/index/index.vue` 的 `onShow()` 和 `getList()` 方法中：
  1. 在檢查 API 響應數據結構之前就訪問了 `res.data.data[0]`
  2. 沒有檢查 `res.data.data` 是否存在或是否為數組
  3. 沒有檢查數組長度是否足夠
  4. `setInterval` 每秒執行一次，導致錯誤重複出現
- **影響**：❌ 運行時錯誤，影響用戶體驗
- **解決方案**：
  1. 添加完整的數據結構檢查：
     ```javascript
     // 修復前（錯誤）
     console.log(res.data.data[0]['price'])
     if (res.data.code == 1) {
         this.btcprice = res.data.data[0].price
     }
     
     // 修復後（正確）
     if (res.data && res.data.code == 1 && res.data.data && Array.isArray(res.data.data) && res.data.data.length >= 3) {
         if (res.data.data[0]) {
             console.log(res.data.data[0]['price'])
             this.btcprice = res.data.data[0].price
         }
     }
     ```
  2. 在訪問數組元素前檢查元素是否存在
  3. 在訪問對象屬性前檢查屬性是否存在
- **已修復的方法**：
  - `onShow()` - 修復了 setInterval 中的 API 請求處理
  - `getList()` - 修復了 API 請求處理
- **避免**：
  1. 訪問數組或對象屬性前，先檢查是否存在
  2. 使用可選鏈操作符 `?.` 或條件判斷
  3. 對 API 響應進行完整的數據驗證

### 問題 9：i18n 數據結構訪問錯誤 ✅ 已解決
- **現象**：控制台出現 TypeError：
  - `Cannot read properties of undefined (reading '0')`
  - 在訪問 i18n 數組屬性時發生
- **原因**：在 `pages/common/login.vue` 中直接訪問 i18n 數組屬性，沒有檢查：
  1. `i18n.common5[6]` - 某些語言包可能沒有索引 6
  2. `i18n.login.placeholder[0]` - 如果 i18n.login 或 placeholder 未定義
  3. `i18n.common[0]` - 如果 i18n.common 未定義
  4. `langsData[0]` - 如果數組為空
- **影響**：❌ 運行時錯誤，影響用戶體驗
- **解決方案**：添加完整的數據結構檢查和安全訪問：
  ```javascript
  // 修復前（錯誤）
  {{i18n.common5[6]}}
  {{i18n.login.placeholder[0]}}
  
  // 修復後（正確）
  {{i18n.common5 && i18n.common5[6] ? i18n.common5[6] : 'Download App'}}
  {{(i18n.login && i18n.login.placeholder && i18n.login.placeholder[0]) ? i18n.login.placeholder[0] : 'Please enter your email address'}}
  ```
- **已修復的位置**：
  - `i18n.common5[6]` - 添加安全檢查
  - `i18n.login.placeholder[0]` 和 `[1]` - 添加安全檢查
  - `i18n.login.label[1]` 和 `[0]` - 添加安全檢查
  - `i18n.login.default[0]` 和 `[1]` - 添加安全檢查
  - `i18n.common[0]` - 添加安全檢查
  - `langsData[0]` - 添加數組長度檢查
- **避免**：
  1. 訪問 i18n 數組屬性前，先檢查對象和數組是否存在
  2. 使用可選鏈操作符 `?.` 或條件判斷
  3. 提供默認值作為後備
