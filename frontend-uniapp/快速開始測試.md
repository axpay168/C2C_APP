## 🔍 調試日誌說明

所有日誌都以 `[REGISTER]` 或 `[LOGIN]` 開頭：

### 註冊相關日誌：
- `[REGISTER] 點擊註冊按鈕` - 用戶點擊註冊
- `[REGISTER] 驗證失敗: 帳號為空` - 驗證失敗
- `[REGISTER] 完整驗證通過，準備提交註冊` - 驗證通過
- `[REGISTER] 準備調用 API` - 準備發送請求
- `[REGISTER] API 響應` - API 返回數據
- `[REGISTER] 註冊成功` - 註冊成功

### 登入相關日誌：
- `[LOGIN] 頁面載入` - 頁面初始化
- `[LOGIN] 點擊登入按鈕` - 用戶點擊登入
- `[LOGIN] 驗證失敗` - 驗證失敗
- `[LOGIN] 開始登入` - 開始 API 調用
- `[LOGIN] API 響應` - API 返回數據
- `[LOGIN] 登入成功` - 登入成功

## 🐛 發現問題時的處理流程

### 1. 立即記錄問題

告訴我：
- 測試項目編號
- 問題描述
- 實際結果 vs 預期結果
- Console 日誌（完整複製）
- Network 請求詳情（如果有）

### 2. 我會立即修復

我會：
- 分析問題根本原因
- 修復代碼
- 告訴您重新測試

### 3. 重新測試

- 按照相同的步驟重新測試
- 確認問題是否已解決
- 如果還有問題，繼續報告
