# 實時測試協作指南

## 測試流程

### 第一步：打開瀏覽器開發者工具

1. 在 HBuilderX 預覽的瀏覽器中，按 `F12` 打開開發者工具
2. 切換到 **Console** 標籤（查看日誌）
3. 切換到 **Network** 標籤（查看 API 請求）

---

## 測試步驟和問題報告格式

### 測試 1：登入頁面載入

**操作**：
1. 打開登入頁面

**請告訴我**：
- ✅ 頁面是否正常顯示？
- ✅ 是否有任何錯誤訊息（紅色文字）？
- ✅ Console 中是否有 `[LOGIN]` 開頭的日誌？

**預期結果**：
- 頁面正常顯示，沒有錯誤
- Console 中可能有初始化相關的日誌

---

### 測試 2：輸入驗證 - 空值測試

**操作**：
1. 不輸入任何內容
2. 直接點擊「登录」按鈕

**請告訴我**：
- ✅ 顯示了什麼錯誤提示？
- ✅ Console 中是否有 `[LOGIN] 驗證失敗` 的日誌？

**預期結果**：
- 顯示：「請輸入帳號」或「Please enter your email」
- Console 顯示：`[LOGIN] 驗證失敗: 帳號為空`

---

### 測試 3：輸入驗證 - 只輸入帳號

**操作**：
1. 輸入帳號：`test@example.com`
2. 不輸入密碼
3. 點擊「登录」

**請告訴我**：
- ✅ 顯示了什麼錯誤提示？
- ✅ Console 中的日誌內容？

**預期結果**：
- 顯示：「請輸入密碼」或「Please enter your password」

---

### 測試 4：登入失敗 - 帳號不存在

**操作**：
1. 輸入帳號：`nonexist@example.com`
2. 輸入密碼：`123456`
3. 點擊「登录」

**請告訴我**：
- ✅ 顯示了什麼錯誤提示？
- ✅ Console 中的 `[LOGIN]` 日誌內容（完整複製）
- ✅ Network 標籤中的 API 請求：
  - 請求 URL
  - 請求方法
  - 請求參數
  - 響應狀態碼
  - 響應數據（完整複製）

**預期結果**：
- 顯示後端返回的錯誤訊息
- Console 顯示完整的 API 響應日誌

---

### 測試 5：登入失敗 - 密碼錯誤

**操作**：
1. 使用存在的帳號
2. 輸入錯誤的密碼
3. 點擊「登录」

**請告訴我**：
- ✅ 顯示了什麼錯誤提示？
- ✅ Console 和 Network 的完整信息

---

### 測試 6：登入成功

**操作**：
1. 輸入正確的帳號和密碼
2. 點擊「登录」

**請告訴我**：
- ✅ 顯示了什麼成功訊息？
- ✅ 是否跳轉到首頁？
- ✅ Console 中是否有 `[LOGIN] 登入成功` 和 `[LOGIN] Token 已保存` 的日誌？
- ✅ Application 標籤 → Local Storage 中是否有 `token` 鍵？

**預期結果**：
- 顯示：「登入成功」或「Login successful」
- 1.2 秒後跳轉到首頁
- Token 保存到本地存儲

---

### 測試 7：語言切換

**操作**：
1. 點擊右上角「中文 | EN」
2. 選擇不同的語言

**請告訴我**：
- ✅ 彈窗是否正常顯示？
- ✅ 選擇語言後，頁面文字是否更新？
- ✅ 當前語言是否高亮顯示？
- ✅ 是否有任何錯誤？

---

### 測試 8：密碼顯示/隱藏

**操作**：
1. 在密碼輸入框輸入密碼
2. 點擊右側眼睛圖標

**請告訴我**：
- ✅ 密碼是否從隱藏變為顯示？
- ✅ 眼睛圖標是否改變？
- ✅ 再次點擊是否恢復隱藏？

---

### 測試 9：記住密碼

**操作**：
1. 勾選「记住密码」
2. 輸入帳號和密碼
3. 登入成功
4. 退出登入
5. 重新打開登入頁面

**請告訴我**：
- ✅ 帳號和密碼是否自動填充？
- ✅ Application → Local Storage 中是否有 `loginAccount` 和 `loginPassword`？

---

### 測試 10：註冊功能 - 空值驗證

**操作**：
1. 點擊「去注册」進入註冊頁面
2. 不輸入任何內容，直接點擊「註冊」

**請告訴我**：
- ✅ 顯示了什麼錯誤提示？
- ✅ Console 中是否有 `[REGISTER]` 日誌？

---

### 測試 11：註冊功能 - 格式驗證

**操作**：
1. 選擇「郵箱註冊」
2. 輸入無效郵箱：`test@`
3. 點擊「註冊」

**請告訴我**：
- ✅ 顯示了什麼錯誤提示？

---

### 測試 12：註冊功能 - 成功註冊

**操作**：
1. 輸入有效郵箱：`newuser@example.com`
2. 輸入密碼：`123456`
3. 輸入確認密碼：`123456`
4. 點擊「註冊」

**請告訴我**：
- ✅ 顯示了什麼訊息？
- ✅ 是否跳轉到登入頁面？
- ✅ Console 和 Network 的完整信息

---

## 問題報告格式

當您發現問題時，請按照以下格式告訴我：

```
【問題編號】測試 X：問題描述

【操作步驟】
1. ...
2. ...

【實際結果】
- 顯示的錯誤訊息：...
- Console 日誌：...
- Network 請求：...

【預期結果】
- 應該：...

【截圖/日誌】
（如果有，請描述或複製）
```

---

## 我將如何修復

當您報告問題後，我會：

1. **立即檢查代碼**：查看相關的函數和邏輯
2. **分析問題原因**：根據日誌和錯誤訊息定位問題
3. **修復代碼**：在 Cursor 中修改相關文件
4. **添加調試日誌**：如果需要，添加更多日誌幫助排查
5. **告訴您重新測試**：修復後請您重新測試

---

## 快速檢查清單

在開始測試前，請確認：

- [ ] 瀏覽器開發者工具已打開（F12）
- [ ] Console 標籤已切換
- [ ] Network 標籤已切換
- [ ] HBuilderX 預覽正在運行
- [ ] 頁面已正常載入

---

## 常見問題快速排查

### 問題：頁面空白或載入失敗

**請告訴我**：
- Console 中的錯誤訊息（完整複製）
- Network 中是否有失敗的請求

### 問題：API 請求失敗

**請告訴我**：
- Network 標籤中的請求詳情
- 響應狀態碼
- 響應數據內容

### 問題：功能不工作

**請告訴我**：
- 具體是哪個功能
- Console 中是否有相關日誌
- 是否有錯誤訊息

---

## 開始測試

現在請按照測試步驟逐一測試，並告訴我每個測試的結果。我會根據您的反饋實時修復問題。

**建議測試順序**：
1. 登入頁面載入
2. 登入功能（錯誤情況 → 成功情況）
3. 語言切換
4. 密碼顯示/隱藏
5. 記住密碼
6. 註冊功能

讓我們開始吧！🚀
