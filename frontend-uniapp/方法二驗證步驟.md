# 方法二：HBuilderX 中安裝依賴 - 驗證步驟

## 安裝步驟回顧

1. 開啟 HBuilderX
2. 開啟專案目錄 `frontend-uniapp`
3. 在 HBuilderX 底部終端執行：`npm install`
4. 等待安裝完成

---

## 驗證方法

### 方法 A：使用驗證腳本（最簡單）

1. **執行驗證腳本**：雙擊 `驗證依賴安裝.bat`
2. **查看結果**：
   - ✓ 所有依賴已正確安裝 → 可以開始編譯
   - ✗ 部分依賴未安裝 → 需要重新執行 `npm install`

### 方法 B：手動檢查目錄

檢查以下目錄是否存在：

```
frontend-uniapp/
├── node_modules/          ← 必須存在
│   ├── echarts/           ← 必須存在
│   ├── ethers/            ← 必須存在
│   ├── tronweb/           ← 必須存在
│   └── vue-i18n/          ← 必須存在
```

**檢查方式**：
1. 在檔案總管中開啟 `frontend-uniapp` 目錄
2. 確認 `node_modules` 資料夾存在
3. 進入 `node_modules` 資料夾
4. 確認以下資料夾存在：
   - `echarts`
   - `ethers`
   - `tronweb`
   - `vue-i18n`

### 方法 C：在 HBuilderX 終端中檢查

在 HBuilderX 底部終端執行：

```bash
# 檢查 node_modules 是否存在
dir node_modules

# 檢查特定依賴
dir node_modules\echarts
dir node_modules\ethers
dir node_modules\tronweb
```

如果看到這些目錄，表示安裝成功。

### 方法 D：檢查 package-lock.json

如果安裝成功，應該會看到：
- `package-lock.json` 檔案存在（或更新時間為剛剛）

---

## 驗證成功後

如果所有依賴都已正確安裝：

1. **在 HBuilderX 中重新載入專案**（可選）：
   - 右鍵專案 → 重新載入專案

2. **執行編譯**：
   - 右鍵專案 → 發行 → 網站-H5

3. **檢查編譯結果**：
   - 應該不會再出現「文件查找失敗」的錯誤
   - 編譯成功後，輸出在 `unpackage/dist/build/h5/`

---

## 如果驗證失敗

### 問題 1：node_modules 不存在

**解決**：
```bash
# 在 HBuilderX 終端中執行
npm install
```

### 問題 2：部分依賴未安裝

**解決**：
```bash
# 清除快取後重新安裝
npm cache clean --force
npm install
```

### 問題 3：npm 命令無法執行

**解決**：
1. 確認 Node.js 已安裝
2. 在 HBuilderX 設定中配置 Node.js 路徑：
   - 工具 → 設定 → 執行時設定 → Node.js 路徑
3. 重新啟動 HBuilderX

### 問題 4：網路問題導致安裝失敗

**解決**：使用國內鏡像
```bash
npm install --registry=https://registry.npmmirror.com
```

- [ ] `node_modules` 目錄存在
- [ ] `node_modules/echarts` 存在
- [ ] `node_modules/ethers` 存在
- [ ] `node_modules/tronweb` 存在
- [ ] `node_modules/vue-i18n` 存在
- [ ] `package-lock.json` 存在（或已更新）
- [ ] HBuilderX 終端中 `npm install` 執行成功且無錯誤

## 下一步

驗證成功後，請在 HBuilderX 中執行：
**發行 → 網站-H5**

如果編譯時仍然出現「文件查找失敗」錯誤，請：
1. 重新載入專案（右鍵專案 → 重新載入專案）
2. 檢查 HBuilderX 是否正確識別專案根目錄
3. 查看 `編譯問題記錄.md` 中的其他解決方案
