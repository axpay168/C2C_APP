## 🚀 HBuilderX H5 預覽設置

### 立即開始預覽

#### 步驟 1：在 HBuilderX 中啟動

1. **確保 HBuilderX 已開啟專案**
   - 專案應顯示在左側專案樹中

2. **點擊「運行」按鈕**
   - 工具欄上的「運行」按鈕
   - 或選單：**運行 → 運行到瀏覽器**

3. **選擇瀏覽器**
   - 選擇 Chrome（推薦）
   - 或其他已安裝的瀏覽器

4. **等待編譯**
   - HBuilderX 會自動啟動開發伺服器
   - 預設端口：`8080`
   - 瀏覽器會自動打開：`http://localhost:8080`

#### 步驟 2：開始開發

1. **在 Cursor 中修改代碼**
   - 打開任何 `.vue`、`.js`、`.css` 文件
   - 進行修改
   - 保存文件（`Ctrl + S`）

2. **自動更新**
   - HBuilderX 會自動檢測文件變更
   - 自動重新編譯
   - 瀏覽器自動刷新顯示最新內容

## 🔧 配置檢查

### HBuilderX 運行配置

#### 檢查項目：
1. **Node.js 路徑**
   - 工具 → 設定 → 執行時設定 → Node.js 路徑
   - 應指向 Node.js 安裝目錄

2. **npm 路徑**
   - 工具 → 設定 → 執行時設定 → npm 路徑
   - 應指向 npm 可執行文件

3. **預設瀏覽器**
   - 工具 → 設定 → 執行時設定 → 瀏覽器
   - 選擇 Chrome 或其他瀏覽器

### 開發伺服器配置

已在 `vue.config.js` 中配置：
```javascript
devServer: {
  host: '0.0.0.0',        // 允許外部訪問
  port: 8080,             // 預設端口
  allowedHosts: 'all',    // 允許所有主機
  disableHostCheck: true  // 禁用主機檢查
}
```

### H5 配置

已在 `manifest.json` 中配置：
```json
"h5": {
  "template": "template.h5.html",
  "router": {
    "mode": "hash",
    "base": "h5/"
  },
  "devServer": {
    "https": false
  }
}
```

## 🆘 如果遇到問題

### 問題 1：無法啟動「運行到瀏覽器」

**解決方案**：
1. 檢查 HBuilderX 的 Node.js 路徑配置
2. 確認 Node.js 已正確安裝
3. 檢查端口 8080 是否被占用

### 問題 2：瀏覽器無法打開

**解決方案**：
1. 檢查防火牆設置
2. 嘗試手動訪問：`http://localhost:8080`
3. 檢查 HBuilderX 的編譯日誌

### 問題 3：修改代碼後沒有更新

**解決方案**：
1. 確認文件已保存
2. 檢查 HBuilderX 是否正在運行
3. 查看 HBuilderX 的編譯日誌
4. 嘗試手動刷新瀏覽器
