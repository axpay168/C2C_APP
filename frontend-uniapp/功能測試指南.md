# 功能測試指南

## 快速測試步驟

### 1. 註冊功能測試

#### 測試錯誤情況：

1. **空值測試**
   - 不輸入任何內容，點擊「註冊」
   - ✅ 應顯示：「请输入邮箱」或「请输入手机号」

2. **郵箱格式錯誤**
   - 選擇「郵箱註冊」
   - 輸入：`test@` 或 `test.com`
   - ✅ 應顯示：「请输入正确的邮箱格式」

3. **手機號格式錯誤**
   - 選擇「手機號註冊」
   - 輸入：`123` 或少於 6 位
   - ✅ 應顯示：「请输入正确的手机号」

4. **密碼長度不足**
   - 輸入密碼：`12345`（少於 6 位）
   - ✅ 應顯示：「密码不能低于6位」

5. **兩次密碼不一致**
   - 密碼：`123456`
   - 確認密碼：`1234567`
   - ✅ 應顯示：「两次密码不一致」

#### 測試成功情況：

6. **成功註冊**
   - 輸入有效郵箱：`test@example.com`
   - 密碼：`123456`
   - 確認密碼：`123456`
   - 點擊「註冊」
   - ✅ 應顯示：「注册成功」
   - ✅ 1.5 秒後跳轉到登入頁面

#### 檢查 API 調用：

打開瀏覽器開發者工具（F12）→ Network 標籤：
- 請求 URL：`/index.php/api/user/register`
- 請求方法：`POST`
- 請求參數：
  ```json
  {
    "username": "test@example.com",
    "password": "123456",
    "invitation_code": "",
    "code": ""
  }
  ```

### 2. 登入功能測試

#### 測試錯誤情況：

1. **帳號為空**
   - 不輸入帳號，點擊「登录」
   - ✅ 應顯示：「Please enter your email」或「请输入邮箱」

2. **密碼為空**
   - 輸入帳號，不輸入密碼
   - ✅ 應顯示：「Please enter your password」或「请输入密码」

3. **帳號不存在**
   - 帳號：`nonexist@example.com`
   - 密碼：`123456`
   - ✅ 應顯示後端返回的錯誤訊息（如：「账号不存在」）

4. **密碼錯誤**
   - 使用存在的帳號，輸入錯誤密碼
   - ✅ 應顯示後端返回的錯誤訊息（如：「密码不正确」）

#### 測試成功情況：

5. **成功登入**
   - 輸入正確的帳號和密碼
   - 點擊「登录」
   - ✅ 應顯示：「登入成功」或「Login successful」
   - ✅ Token 保存到本地存儲
   - ✅ 1.2 秒後跳轉到首頁

#### 檢查 API 調用：

打開瀏覽器開發者工具（F12）→ Network 標籤：
- 請求 URL：`/index.php/api/user/login`
- 請求方法：`POST`
- 請求參數：
  ```json
  {
    "account": "test@example.com",
    "password": "123456"
  }
  ```

#### 檢查響應數據：

成功響應：
```json
{
  "code": 1,
  "data": {
    "userinfo": {
      "token": "xxx...",
      // ... 其他用戶信息
    }
  }
}
```

失敗響應：
```json
{
  "code": 0,
  "msg": "账号不存在"
}
```

### 3. 語言切換功能測試

1. **打開語言選擇**
   - 點擊右上角「中文 | EN」
   - ✅ 應彈出語言選擇彈窗

2. **切換語言**
   - 選擇「English」或「中文」
   - ✅ 頁面文字應更新為對應語言
   - ✅ 當前語言應高亮顯示

3. **檢查文字更新**
   - 登入按鈕文字應更新
   - 輸入框提示文字應更新
   - 其他頁面文字應更新

---

### 4. 密碼顯示/隱藏功能測試

1. **輸入密碼**
   - 在密碼輸入框輸入：`123456`
   - ✅ 應顯示為：`••••••`（隱藏狀態）

2. **顯示密碼**
   - 點擊右側眼睛圖標
   - ✅ 應顯示為：`123456`（明文狀態）
   - ✅ 眼睛圖標應改變

3. **隱藏密碼**
   - 再次點擊眼睛圖標
   - ✅ 應顯示為：`••••••`（隱藏狀態）

---

### 5. 記住密碼功能測試

1. **勾選記住密碼**
   - 勾選「记住密码」複選框
   - 輸入帳號和密碼
   - 登入成功
   - ✅ 複選框應顯示勾選標記（✓）

2. **檢查存儲**
   - 打開瀏覽器開發者工具（F12）→ Application 標籤 → Local Storage
   - ✅ 應有 `loginAccount` 和 `loginPassword` 鍵值對

3. **重新打開登入頁面**
   - 退出登入
   - 重新打開登入頁面
   - ✅ 帳號和密碼應自動填充

4. **不勾選記住密碼**
   - 不勾選「记住密码」
   - 登入成功
   - ✅ 不應保存帳號和密碼
   - ✅ 重新打開時帳號和密碼為空

## 常見問題排查

### 問題 1：API 請求失敗

**症狀**：顯示「網路請求失敗」或「無法連接伺服器」

**排查步驟**：
1. 檢查網路連接
2. 檢查 API 基礎 URL 是否正確
3. 檢查瀏覽器控制台是否有 CORS 錯誤
4. 檢查後端服務是否運行

### 問題 2：登入後沒有跳轉

**症狀**：登入成功但沒有跳轉到首頁

**排查步驟**：
1. 檢查 token 是否正確保存
2. 檢查 `uni.reLaunch` 是否執行
3. 檢查首頁路由是否正確

### 問題 3：語言切換不生效

**症狀**：切換語言後文字沒有更新

**排查步驟**：
1. 檢查語言包是否存在對應的鍵值
2. 檢查 `i18n` 配置是否正確
3. 檢查 `setLang` 方法是否正確執行

### 問題 4：記住密碼不生效

**症狀**：勾選記住密碼後重新打開沒有自動填充

**排查步驟**：
1. 檢查 `uni.setStorageSync` 是否執行
2. 檢查 `onLoad` 中是否正確讀取存儲
3. 檢查瀏覽器是否允許本地存儲
