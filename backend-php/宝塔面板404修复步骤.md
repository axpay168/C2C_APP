# 宝塔面板404错误修复步骤

## 🎯 问题

访问 `http://MXTRX.TOP/buSRxMqJOo.php/index/login` 显示 **404 Not Found nginx**

## ✅ 快速修复（3步）

### 第一步：检查网站根目录

1. **登录宝塔面板**
2. **点击左侧菜单"网站"**
3. **找到域名 `MXTRX.TOP` 的网站**
4. **点击右侧的"设置"按钮**
5. **进入"网站目录"标签**

**关键配置**：
- **网站目录**：必须设置为 `/www/wwwroot/hkex-sm.com/public` ⚠️ **重要**
- **运行目录**：设置为 `/public` 或留空

**常见错误**：
- ❌ `/www/wwwroot/hkex-sm.com/` （缺少 `/public`）
- ✅ `/www/wwwroot/hkex-sm.com/public` （正确）

### 第二步：检查Nginx配置

1. **在网站设置中，点击"配置文件"标签**
2. **查找 `root` 指令**，应该显示：
   ```nginx
   root /www/wwwroot/hkex-sm.com/public;
   ```
3. **如果显示的是 `/www/wwwroot/hkex-sm.com/`，请修改为 `/www/wwwroot/hkex-sm.com/public`**
4. **检查 `location ~ \.php$` 配置**，应该类似：
   ```nginx
   location ~ \.php$ {
       fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 根据PHP版本调整
       fastcgi_index index.php;
       fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
       include fastcgi_params;
   }
   ```
5. **点击"保存"按钮**

### 第三步：重载配置

1. **点击网站右侧的"重载"或"重启"按钮**
2. **或进入"软件商店" → "Nginx" → "设置" → "服务" → "重载配置"**

## 🔍 详细检查清单

### 检查项1：网站根目录
- [ ] 网站目录 = `/www/wwwroot/hkex-sm.com/public`
- [ ] 运行目录 = `/public` 或留空

### 检查项2：Nginx配置
- [ ] `root` 指令指向 `/www/wwwroot/hkex-sm.com/public`
- [ ] `location ~ \.php$` 配置存在
- [ ] `fastcgi_pass` 路径正确（根据PHP版本）

### 检查项3：PHP配置
- [ ] PHP版本已选择（建议8.2）
- [ ] PHP-FPM服务正在运行

### 检查项4：文件存在
- [ ] `public/buSRxMqJOo.php` 文件存在
- [ ] `public/index.php` 文件存在

## 📸 配置截图参考

### 网站目录配置应该显示：
```
网站目录: /www/wwwroot/hkex-sm.com/public
运行目录: /public
```

### Nginx配置文件中应该包含：
```nginx
server {
    listen 80;
    server_name MXTRX.TOP;
    
    root /www/wwwroot/hkex-sm.com/public;  # ← 关键：必须指向public目录
    index index.php index.html;
    
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    
    location ~ \.php$ {
        fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 根据PHP版本调整
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

## 🐛 如果仍然404

### 检查1：查看错误日志
1. 在网站设置中，点击"日志"标签
2. 查看"错误日志"
3. 查找相关错误信息

### 检查2：测试PHP文件
1. 在 `public/` 目录创建 `test.php`：
   ```php
   <?php echo "PHP works!"; ?>
   ```
2. 访问：`http://MXTRX.TOP/test.php`
3. 如果显示 "PHP works!"，说明PHP配置正确
4. 如果404，说明网站根目录配置错误

### 检查3：确认PHP版本和sock路径
1. 在网站设置中，查看"PHP版本"
2. 确认PHP版本（如8.2）
3. 在Nginx配置中，确认 `fastcgi_pass` 路径：
   - PHP 8.2: `unix:/tmp/php-cgi-82.sock`
   - PHP 8.1: `unix:/tmp/php-cgi-81.sock`
   - PHP 7.4: `unix:/tmp/php-cgi-74.sock`

## ⚡ 快速命令检查（SSH权限）

如果有SSH权限，可以执行：

```bash

# 查看Nginx错误日志
tail -20 /www/wwwlogs/MXTRX.TOP.error.log

# 重启Nginx

```
