```bash

# 执行以下SQL命令

CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'hkex02'@'localhost' IDENTIFIED BY 'cZBYPH2XD4YM3yfX';
GRANT ALL PRIVILEGES ON `hkex02`.* TO 'hkex02'@'localhost';
```

## 🔧 故障排查

### 问题1：导入时提示"文件过大"

**解决方案**：
1. 修改phpMyAdmin上传文件大小限制
2. 或使用命令行导入（需要SSH权限）
3. 或分割SQL文件

### 问题2：导入后表不存在

**解决方案**：
1. 检查SQL文件是否完整
2. 检查数据库字符集设置（应为utf8mb4）
3. 查看导入日志中的错误信息
4. 确认导入时选择了正确的数据库

### 问题3：用户权限不足

**解决方案**：
```sql
-- 使用root用户执行
GRANT ALL PRIVILEGES ON `hkex02`.* TO 'hkex02'@'localhost';
FLUSH PRIVILEGES;
```

### 问题4：字符集问题

**解决方案**：
确保数据库字符集为 `utf8mb4`：
```sql
ALTER DATABASE `hkex02` CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
