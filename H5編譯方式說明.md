# H5 前端編譯方式說明

## 🔍 為什麼需要重新編譯？

**重要**：修改了源代碼（`.vue` 文件）後，**必須重新編譯**才能生效！

- **源代碼**：`frontend-uniapp/src/pages/common/register.vue`（Vue 組件，瀏覽器無法直接運行）
- **編譯產物**：`backend-php/public/h5/static/js/pages-common-register.xxx.js`（編譯後的 JavaScript，瀏覽器可以運行）

**寶塔面板的網站開啟/關閉**：
- ❌ **不會重新編譯代碼**
- ✅ 只會重啟 Nginx 服務
- ✅ 可以清除 Nginx 緩存
- ✅ 可以重新加載 Nginx 配置

## 📋 編譯方式對比

### 方式 1：命令行編譯（推薦，最直接）

```bash
cd /www/wwwroot/old/frontend-uniapp
npm run build:h5
cp -r dist/build/h5/* ../backend-php/public/h5/
```

**優點**：
- ✅ 最快速、最直接
- ✅ 可以自動化
- ✅ 適合服務器環境

**缺點**：
- ❌ 需要 SSH 訪問服務器
- ❌ 需要 Node.js 環境

---

### 方式 2：使用 HBuilderX 圖形界面（適合不熟悉命令行的用戶）

**步驟**：

1. **下載並安裝 HBuilderX**
   - 訪問：https://www.dcloud.io/hbuilderx.html
   - 下載並安裝

2. **打開項目**
   - 在 HBuilderX 中選擇「文件」→「打開目錄」
   - 選擇：`/www/wwwroot/old/frontend-uniapp`

3. **編譯 H5**
   - 點擊頂部菜單「發行」→「網站-H5」
   - 等待編譯完成
   - 編譯產物會在：`unpackage/dist/build/h5/`

4. **複製到服務器**
   - 使用寶塔面板的「文件管理」
   - 將 `unpackage/dist/build/h5/` 目錄下的所有文件
   - 上傳/複製到 `/www/wwwroot/old/backend-php/public/h5/`

**優點**：
- ✅ 圖形界面，操作簡單
- ✅ 不需要命令行知識
- ✅ 可以本地編譯後上傳

**缺點**：
- ❌ 需要下載安裝 HBuilderX
- ❌ 需要手動上傳文件

---

### 方式 3：使用寶塔面板（部分操作）

**寶塔面板可以做的**：

1. **文件管理**：
   - ✅ 可以上傳編譯好的文件
   - ✅ 可以複製/移動文件
   - ✅ 可以刪除舊文件

2. **網站管理**：
   - ✅ 可以重啟 Nginx（清除緩存）
   - ✅ 可以清除網站緩存
   - ✅ 可以查看錯誤日誌

**寶塔面板不能做的**：
- ❌ **不能編譯 Vue 源代碼**
- ❌ 不能執行 `npm run build:h5`

---

## 🎯 推薦流程

### 如果您熟悉命令行：

```bash
# 1. 編譯
cd /www/wwwroot/old/frontend-uniapp
npm run build:h5

# 2. 複製
cp -r dist/build/h5/* ../backend-php/public/h5/

# 3. 在寶塔面板中重啟 Nginx（可選，清除緩存）
```

### 如果您不熟悉命令行：

1. **使用 HBuilderX 編譯**（本地電腦）
2. **使用寶塔面板上傳文件**（文件管理 → 上傳）
3. **在寶塔面板中重啟 Nginx**（網站 → 設置 → 重載配置）

---

## ⚠️ 重要提示

### 什麼時候需要重新編譯？

- ✅ **修改了 `.vue` 源代碼文件**（如 `register.vue`）
- ✅ **修改了 `common/http.api.js` 等配置文件**
- ✅ **修改了樣式文件**
- ❌ **只修改了 Nginx 配置** → 只需重啟 Nginx
- ❌ **只修改了 PHP 後端代碼** → 不需要重新編譯前端

### 編譯後還需要做什麼？

1. **複製編譯產物**到 `public/h5/` 目錄
2. **清除瀏覽器緩存**（Ctrl+Shift+Delete 或使用無痕模式）
3. **可選：重啟 Nginx**（在寶塔面板中操作）

---

## 🔧 寶塔面板操作步驟

### 如果使用 HBuilderX 編譯後上傳：

1. **打開寶塔面板** → **文件管理**
2. **進入目錄**：`/www/wwwroot/old/backend-php/public/h5/`
3. **刪除舊文件**（可選，建議先備份）
4. **上傳新文件**：
   - 點擊「上傳」
   - 選擇編譯好的 `h5` 目錄下的所有文件
   - 等待上傳完成
5. **重啟 Nginx**（可選）：
   - 進入「網站」→ 找到對應網站
   - 點擊「設置」→「重載配置」

---

## 📝 總結

| 操作 | 命令行 | HBuilderX | 寶塔面板 |
|------|--------|-----------|----------|
| **編譯源代碼** | ✅ 可以 | ✅ 可以 | ❌ 不可以 |
| **上傳文件** | ✅ 可以 | ❌ 不可以 | ✅ 可以 |
| **重啟 Nginx** | ✅ 可以 | ❌ 不可以 | ✅ 可以 |
| **清除緩存** | ✅ 可以 | ❌ 不可以 | ✅ 可以 |

**結論**：
- **編譯必須用命令行或 HBuilderX**（寶塔面板無法編譯）
- **上傳和重啟可以用寶塔面板**（更方便）

---

**建議**：如果您不熟悉命令行，使用 **HBuilderX 編譯 + 寶塔面板上傳** 是最簡單的方式！
