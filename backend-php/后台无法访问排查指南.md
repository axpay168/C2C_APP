# 后台无法访问排查指南

访问地址：`http://MXTRX.TOP/buSRxMqJOo.php/index/login`

## 🔍 快速排查步骤

### 1. 检查Web服务器状态

#### 在宝塔面板中检查：
1. 登录宝塔面板
2. 查看"软件商店" → "已安装"
3. 确认Nginx或Apache是否已安装并运行
4. 如果未运行，点击"启动"

#### 命令行检查（如果有SSH权限）：
```bash
# 检查Nginx
systemctl status nginx
# 或
systemctl status apache2
```

### 2. 检查网站配置

#### 在宝塔面板中：
1. 进入"网站"管理
2. 找到域名 `MXTRX.TOP` 的网站
3. 点击"设置"
4. 检查以下配置：

**重要配置项**：
- **网站目录**：必须设置为 `/www/wwwroot/hkex-sm.com/public`
- **运行目录**：设置为 `/public` 或留空
- **PHP版本**：选择 PHP 7.2 或更高版本（建议 PHP 8.2）

### 3. 检查PHP-FPM状态

在宝塔面板中：
1. 进入"软件商店" → "PHP"
2. 找到对应的PHP版本
3. 点击"设置" → "服务"
4. 确认PHP-FPM服务状态为"运行中"

### 4. 检查文件权限

确保以下目录有正确的权限：
```bash
# 在宝塔面板"文件"管理中，检查以下目录权限：
- runtime/ 目录：755 或 777
- public/uploads/ 目录：755 或 777
- public/buSRxMqJOo.php 文件：644
```

### 5. 检查数据库连接

**可能原因**：数据库未导入或连接失败

#### 检查数据库是否已导入：
1. 登录宝塔面板
2. 进入"数据库"管理
3. 找到数据库 `hkex02`

如果没有表，需要先导入数据库（参考 `宝塔面板导入数据库指南.md`）

#### 检查数据库配置：
确认 `application/database.php` 中的配置：
```php

'username' => 'hkex02',
'password' => 'cZBYPH2XD4YM3yfX',
```

### 6. 查看错误日志

#### 在宝塔面板中查看：

## 🛠️ 常见错误及解决方案

### 错误1: 404 Not Found
**原因**：网站根目录配置错误或URL重写未生效

**解决方案**：
1. 确认网站根目录为 `/www/wwwroot/hkex-sm.com/public`
2. 如果是Nginx，检查配置文件中是否有 `try_files` 指令
3. 如果是Apache，确认已启用 `mod_rewrite` 模块

### 错误2: 500 Internal Server Error
**原因**：PHP错误、数据库连接失败、文件权限问题

**解决方案**：
1. 查看PHP错误日志
2. 检查数据库连接配置
3. 检查文件权限
4. 检查 `runtime/` 目录是否有写入权限

### 错误3: 502 Bad Gateway
**原因**：PHP-FPM未运行或配置错误

**解决方案**：
1. 重启PHP-FPM服务
2. 检查PHP-FPM配置
3. 确认PHP版本与网站配置一致

### 错误4: 数据库连接错误
**原因**：数据库未导入、配置错误、MySQL服务未运行

**解决方案**：
1. 确认数据库 `hkex02` 已创建
2. 确认已导入SQL文件
3. 检查数据库配置是否正确
4. 确认MySQL服务正在运行

### 错误5: 空白页面
**原因**：PHP错误被隐藏、内存不足、超时

**解决方案**：
1. 开启PHP错误显示（临时调试）
2. 查看PHP错误日志
3. 增加PHP内存限制和执行时间

## 🔧 快速修复命令（SSH权限）

如果有SSH权限，可以执行以下命令：

```bash
# 1. 检查Web服务器状态
systemctl status nginx
# 或
systemctl status apache2

# 2. 重启Web服务器
systemctl restart nginx
# 或
systemctl restart apache2

# 3. 重启PHP-FPM
systemctl restart php-fpm-82
# 或对应版本的PHP-FPM

# 4. 设置文件权限
chmod -R 755 /www/wwwroot/hkex-sm.com/runtime/
chmod -R 755 /www/wwwroot/hkex-sm.com/public/uploads/

# 5. 检查PHP配置
php -v
php -m | grep -E "pdo_mysql|curl|json|bcmath"
```
