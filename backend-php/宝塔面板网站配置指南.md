# 宝塔面板网站配置指南

## 🎯 配置目标

确保网站 `MXTRX.TOP` 能正常访问后台：`http://MXTRX.TOP/buSRxMqJOo.php/index/login`

## 📋 详细配置步骤

### 第一步：创建网站（如果尚未创建）

1. 登录宝塔面板
2. 点击左侧菜单 **"网站"**
3. 点击 **"添加站点"**
4. 填写信息：
   - **域名**：`MXTRX.TOP`
   - **备注**：HKEX项目
   - **根目录**：`/www/wwwroot/hkex-sm.com/public` ⚠️ **重要**
   - **FTP**：不创建（或根据需要创建）
   - **数据库**：MySQL，选择 `hkex02` 数据库
   - **PHP版本**：选择 PHP 8.2（或 7.2+）

5. 点击 **"提交"**

### 第二步：配置网站设置

1. 在网站列表中，找到 `MXTRX.TOP`
2. 点击右侧的 **"设置"** 按钮
3. 进入 **"网站目录"** 标签：

**关键配置**：
- ✅ **网站目录**：`/www/wwwroot/hkex-sm.com/public`
- ✅ **运行目录**：选择 `/public` 或留空
- ✅ **防跨站攻击(open_basedir)**：关闭（或添加项目根目录到白名单）

4. 进入 **"默认文档"** 标签：
   - 确保包含：`index.php`, `index.html`

5. 进入 **"PHP版本"** 标签：
   - 选择 PHP 8.2（或 7.2+）
   - 点击 **"切换"**

### 第三步：配置PHP设置

1. 进入 **"软件商店"** → **"PHP"** → 找到对应版本
2. 点击 **"设置"**
3. 进入 **"安装扩展"** 标签，确保已安装：
   - ✅ pdo_mysql
   - ✅ curl
   - ✅ json
   - ✅ bcmath
   - ✅ mysqli

4. 进入 **"性能调整"** 标签（可选优化）：
   - **max_execution_time**：300
   - **memory_limit**：256M 或更高
   - **post_max_size**：50M
   - **upload_max_filesize**：50M

5. 进入 **"服务"** 标签：
   - 确认 **PHP-FPM** 状态为 **"运行中"**
   - 如果未运行，点击 **"启动"**

### 第四步：配置Nginx（如果使用Nginx）

1. 在网站设置中，进入 **"配置文件"** 标签
2. 确认配置类似以下内容：

```nginx
server {
    listen 80;
    server_name MXTRX.TOP;
    root /www/wwwroot/hkex-sm.com/public;
    index index.php index.html;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 根据PHP版本调整
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.ht {
        deny all;
    }
}
```

3. 点击 **"保存"**

### 第五步：配置Apache（如果使用Apache）

1. 确认已启用 `mod_rewrite` 模块：
   - 进入 **"软件商店"** → **"Apache"** → **"设置"**
   - 进入 **"模块"** 标签
   - 确认 `mod_rewrite` 已启用

2. 在网站设置中，确认 `.htaccess` 文件存在：
   - 路径：`/www/wwwroot/hkex-sm.com/public/.htaccess`
   - 内容应包含URL重写规则

### 第六步：设置文件权限

1. 进入 **"文件"** 管理
2. 导航到 `/www/wwwroot/hkex-sm.com/`
3. 设置目录权限：
   - `runtime/` → 右键 → **"权限"** → 设置为 `755`
   - `public/uploads/` → 右键 → **"权限"** → 设置为 `755`

### 第七步：验证配置

1. **测试PHP**：
   - 在 `public/` 目录创建 `test.php`：
     ```php
     <?php phpinfo(); ?>
     ```
   - 访问：`http://MXTRX.TOP/test.php`
   - 应该能看到PHP信息页面

2. **测试后台**：
   - 访问：`http://MXTRX.TOP/buSRxMqJOo.php/index/login`
   - 应该能看到登录页面

3. **查看日志**：
   - 如果出错，查看网站错误日志
   - 路径：网站设置 → **"日志"** 标签

## ⚠️ 常见配置错误

### 错误1：网站根目录错误
❌ 错误：`/www/wwwroot/hkex-sm.com/`
✅ 正确：`/www/wwwroot/hkex-sm.com/public`

### 错误2：运行目录未设置
如果设置了运行目录，确保是 `/public`

### 错误3：PHP版本过低
确保使用 PHP 7.2 或更高版本

### 错误4：防跨站攻击未关闭
如果开启，需要将项目根目录添加到白名单

## 🔍 故障排查

如果配置后仍无法访问：

1. **检查网站状态**：确认网站为"运行中"
2. **检查PHP-FPM**：确认PHP-FPM服务运行中
3. **查看错误日志**：网站设置 → 日志 → 错误日志
4. **检查数据库**：确认数据库已导入
5. **测试PHP**：创建test.php测试PHP是否正常
