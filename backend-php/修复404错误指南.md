# 修复404错误指南

## 🔍 问题诊断

访问 `http://MXTRX.TOP/buSRxMqJOo.php/index/login` 显示 **404 Not Found nginx**

这说明：
- ✅ Nginx正在运行
- ❌ 网站配置不正确
- ❌ 网站根目录可能指向错误

## 🎯 快速解决方案

### 在宝塔面板中操作

#### 第一步：检查网站配置

1. **登录宝塔面板**
2. **进入"网站"管理**
3. **找到域名 `MXTRX.TOP` 的网站**
4. **点击右侧的"设置"按钮**

#### 第二步：修正网站根目录

**关键配置**：
- **网站目录**：必须设置为 `/www/wwwroot/hkex-sm.com/public` ⚠️ **重要**
- **运行目录**：设置为 `/public` 或留空
- **PHP版本**：选择 PHP 8.2（或 7.2+）

**常见错误**：
- ❌ 错误：`/www/wwwroot/hkex-sm.com/`
- ✅ 正确：`/www/wwwroot/hkex-sm.com/public`

#### 第三步：检查Nginx配置

1. 在网站设置中，点击 **"配置文件"** 标签
2. 确认配置包含以下内容：

```nginx
root /www/wwwroot/hkex-sm.com/public;
index index.php index.html;

location / {
    try_files $uri $uri/ /index.php?$query_string;
}

location ~ \.php$ {
    fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 根据PHP版本调整
    fastcgi_index index.php;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    include fastcgi_params;
}
```

3. 如果配置不正确，点击 **"保存"** 并 **"重载配置"**

#### 第四步：重启服务

1. 点击网站右侧的 **"重启"** 按钮
2. 或进入 **"软件商店"** → **"Nginx"** → **"设置"** → **"服务"** → **"重启"**

## 🔧 详细配置步骤

### 方法一：通过宝塔面板修改（推荐）

1. **网站设置** → **"网站目录"** 标签：
   - 网站目录：`/www/wwwroot/hkex-sm.com/public`
   - 运行目录：`/public` 或留空

2. **网站设置** → **"配置文件"** 标签：
   - 检查 `root` 指令是否正确
   - 检查 `location ~ \.php$` 配置是否正确
   - 检查 `fastcgi_pass` 路径是否正确（根据PHP版本）

3. **保存并重载配置**

### 方法二：直接编辑Nginx配置文件

如果有SSH权限：

1. **找到配置文件**：
```bash
# 宝塔面板Nginx配置文件通常在：
/www/server/panel/vhost/nginx/MXTRX.TOP.conf
# 或根据网站ID：
/www/server/panel/vhost/nginx/你的网站ID.conf
```

2. **编辑配置文件**：
```bash
nano /www/server/panel/vhost/nginx/MXTRX.TOP.conf
```

3. **确认配置**：
```nginx
server {
    listen 80;
    server_name MXTRX.TOP;
    
    # 关键：必须指向public目录
    root /www/wwwroot/hkex-sm.com/public;
    index index.php index.html;
    
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    
    location ~ \.php$ {
        fastcgi_pass unix:/tmp/php-cgi-82.sock;  # 根据PHP版本调整
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

4. **测试配置**：
```bash
nginx -t
```

5. **重载配置**：
```bash
nginx -s reload
# 或
systemctl reload nginx
```

## 🔍 检查PHP-FPM配置

### 确认PHP版本和sock路径

1. **在宝塔面板中**：
   - 进入 **"网站"** → **"设置"** → **"PHP版本"**
   - 查看当前使用的PHP版本

2. **确认sock路径**：
   - PHP 8.2: `/tmp/php-cgi-82.sock`
   - PHP 8.1: `/tmp/php-cgi-81.sock`
   - PHP 7.4: `/tmp/php-cgi-74.sock`

3. **在Nginx配置中使用正确的sock路径**

## 🐛 常见问题

### 问题1：仍然404

**检查**：
1. 确认文件是否存在：`ls -la /www/wwwroot/hkex-sm.com/public/buSRxMqJOo.php`
2. 确认网站根目录配置
3. 查看Nginx错误日志：`/www/wwwlogs/MXTRX.TOP.error.log`

### 问题2：502 Bad Gateway

**原因**：PHP-FPM未运行或sock路径错误

**解决**：
1. 重启PHP-FPM服务
2. 检查sock路径是否正确

### 问题3：403 Forbidden

**原因**：文件权限问题

**解决**：
```bash
chmod 644 /www/wwwroot/hkex-sm.com/public/buSRxMqJOo.php
chmod 755 /www/wwwroot/hkex-sm.com/public/
```
