# old 專案 - 新伺服器部署說明（一看即懂）

## 一、專案結構概覽

```
old/
├── backend-php/         # PHP 後端（FastAdmin / ThinkPHP 5）
│   ├── application/     # 業務邏輯、配置
│   ├── public/          # 入口、H5 前端、靜態資源
│   └── config/          # 系統配置
├── frontend-uniapp/     # uni-app 前端源碼（Vue 2）
├── db/                  # 數據庫
│   ├── schema_full.sql  # 建表結構
│   ├── old_full.sql     # 完整數據（結構 + 數據）
│   └── README.md        # 表說明
├── docs/                # 文檔
├── scripts/             # 腳本
└── 部署說明.md          # 本文件
```

## 二、部署步驟（順序執行）

### 1. 克隆專案

```bash
cd /www/wwwroot
git clone git@github.com:axpay168/old.git
cd old
```

### 2. 建立與導入數據庫

```bash
# 建立數據庫與用戶（密碼請自行修改）
mysql -u root -p -e "
CREATE DATABASE IF NOT EXISTS old DEFAULT CHARSET utf8mb4;
CREATE USER IF NOT EXISTS 'old'@'localhost' IDENTIFIED BY 'fRiTc2bjFtBbSMHC';
GRANT ALL ON old.* TO 'old'@'localhost';
FLUSH PRIVILEGES;
"

# 導入完整數據（結構 + 數據）
mysql -u old -p'fRiTc2bjFtBbSMHC' old < db/old_full.sql
```

> 若使用寶塔，可在「數據庫」中新建庫 `old`、用戶 `old`，再導入 `db/old_full.sql`。

### 3. 後端依賴與權限

```bash
cd backend-php
composer install
chmod -R 755 runtime
chmod -R 755 public/uploads
chmod -R 755 application/runtime
```

### 4. 修改數據庫配置（若與預設不同）

編輯 `backend-php/application/database.php`，確認：

- `database`：`old`
- `username`：`old`
- `password`：`fRiTc2bjFtBbSMHC`
- `prefix`：`fa_`

或通過環境變量配置（建議生產環境使用 .env）。

### 5. Nginx 站點配置

- **根目錄**：指向 `old/backend-php/public`
- **後台入口**：需支援 pathinfo，例如 `/index.php/admin/index/index`
- **後台自訂入口**：若使用 `buSRxMqJOo.php`，需確保該文件名被 Nginx 正確處理

參考配置片段：

```nginx
root /www/wwwroot/old/backend-php/public;
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php?s=$1 last;
        break;
    }
}
location ~ \.php$ {
    fastcgi_split_path_info ^(.+\.php)(/.+)$;
    fastcgi_param PATH_INFO $fastcgi_path_info;
    # ... fastcgi_pass 等
}
```

### 6. 前端 H5 編譯與部署（可選）

若需要從源碼重新編譯 H5：

```bash
cd frontend-uniapp
npm install --legacy-peer-deps
# 方式 A：npm 編譯（可能有 recyclableRender 錯誤）
UNI_CLI_CONTEXT=$(pwd) UNI_INPUT_DIR=$(pwd) npm run build:h5
# 方式 B：建議用 HBuilderX 發行 → 網站-H5，輸出到 unpackage/dist/build/h5/
cp -r unpackage/dist/build/h5/* ../backend-php/public/h5/
```

若已有 `static-h5/` 或編譯產物，可直接複製到 `backend-php/public/h5/`。

### 7. API 域名 / 環境變量

前端 API 基礎地址在 `frontend-uniapp/config/` 或 store 中配置。  
部署到新域名後，需修改為新域名的 API 地址（如 `https://新域名/api/`）。

## 四、驗證

1. 訪問 `https://您的域名/`，確認 H5 首頁可打開。
2. 訪問 `https://您的域名/index.php` 或後台入口，確認後台可登入。
3. 測試 API 調用（登錄、交易等）是否正常。
