## 🐛 問題列表

### 問題 #001：buy.vue 中訪問 undefined.length 導致運行時錯誤

**發現時間**：當前  
**測試項目**：buy.vue 頁面載入  
**問題描述**：頁面載入時出現 `TypeError: Cannot read property 'length' of undefined`  
**錯誤訊息**：
```
[Vue warn]: Error in render: "TypeError: Cannot read property 'length' of undefined"
found in
---> at pages/index/buy.vue
```

**根本原因分析**：
- 第 20 行：`v-if="userinfo.bank.length > 0"` 中，`userinfo.bank` 可能是 `undefined`
- `userinfo` 初始化為空數組 `[]`，但 `userinfo.bank` 屬性不存在
- 當 API 還未返回數據時，嘗試訪問 `undefined.length` 導致錯誤

**修復方案**：
1. 修改 `v-if` 條件，添加完整的安全檢查
2. 修改 `userinfo` 初始化，確保 `bank` 屬性存在
3. 在 `getuserinfo()` 方法中添加錯誤處理和數據驗證
4. 添加對 `userinfo.money` 和 `userinfo.bank[0]` 的安全訪問

**修復狀態**：✅ 已修復

**修復內容**：
- ✅ 修改 `v-if` 條件為：`userinfo && userinfo.bank && Array.isArray(userinfo.bank) && userinfo.bank.length > 0`
- ✅ 修改 `userinfo` 初始化為對象：`{ bank: [], username: '', money: '' }`
- ✅ 在 `getuserinfo()` 中添加錯誤處理和數據驗證
- ✅ 添加對所有 `userinfo` 屬性的安全訪問
- ✅ 修復 `getrobot()` 方法，添加安全檢查

**測試結果**：待測試
