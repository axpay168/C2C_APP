# 宝塔面板重新建立数据库指南

## 🎯 目标

重新建立 `hkex02` 数据库，删除旧数据并导入新的SQL文件。

## 📋 操作步骤

### 第一步：删除现有数据库

1. **登录宝塔面板**
2. **点击左侧菜单"数据库"**
3. **找到数据库 `hkex02`**
4. **点击右侧的"删除"按钮**
5. **确认删除**（会同时删除数据库和用户）

### 第二步：创建新数据库

1. **在"数据库"页面，点击"添加数据库"**
2. **填写信息**：
   - **数据库名**: `hkex02`
   - **用户名**: `hkex02`
   - **密码**: `cZBYPH2XD4YM3yfX`
   - **访问权限**: `本地服务器`
   - **字符集**: `utf8mb4`
3. **点击"提交"**

### 第三步：导入SQL文件

#### 方法A：通过数据库管理界面导入（推荐）

1. **在数据库列表中找到 `hkex02`**
2. **点击右侧的"导入"按钮**
3. **点击"选择文件"或"上传文件"**
4. **选择第一个SQL文件**：
   - `数据库前端/hkex_sm_com_20240321_230819.sql`
5. **点击"开始导入"**
6. **等待导入完成后，再次点击"导入"**
7. **选择第二个SQL文件**：
   - `数据库前端/hkex_sm_com_20240321_231454.sql`
8. **点击"开始导入"**

#### 方法B：通过phpMyAdmin导入

1. **找到数据库 `hkex02`**
2. **点击右侧的"管理"或"phpMyAdmin"按钮**
3. **在左侧选择数据库 `hkex02`**
4. **点击顶部菜单的"导入"标签**
5. **点击"选择文件"**，选择第一个SQL文件
6. **点击"执行"** 开始导入
7. **重复步骤4-6导入第二个SQL文件**

### 第四步：验证导入

导入完成后，在phpMyAdmin中检查：

1. **查看表列表**：
   ```sql
   SHOW TABLES LIKE 'fa_%';
   ```
   应该能看到很多以 `fa_` 开头的表

2. **检查关键表**：
   ```sql
   SELECT COUNT(*) FROM fa_admin;
   SELECT COUNT(*) FROM fa_config;
   SELECT COUNT(*) FROM fa_user;
   ```

3. **检查管理员账号**：
   ```sql
   SELECT id, username, nickname, email, status FROM fa_admin WHERE username='admin';
   ```
   应该能看到管理员账号

## 🔧 如果导入失败

### 问题1：导入时提示"文件过大"

**解决方案**：
1. 修改phpMyAdmin上传文件大小限制
2. 或使用命令行导入（需要SSH权限）
3. 或分割SQL文件

### 问题2：导入后表不存在

**解决方案**：
1. 检查SQL文件是否完整
2. 检查数据库字符集设置（应为utf8mb4）
3. 查看导入日志中的错误信息
4. 确认导入时选择了正确的数据库

### 问题3：用户权限不足

**解决方案**：
在phpMyAdmin中执行：
```sql
GRANT ALL PRIVILEGES ON `hkex02`.* TO 'hkex02'@'localhost';
FLUSH PRIVILEGES;
```

### 问题4：字符集问题

**解决方案**：
确保数据库字符集为 `utf8mb4`：
```sql
ALTER DATABASE `hkex02` CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
